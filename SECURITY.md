# 安全政策

## 支持范围

云粘贴目前优先为最新发布版本提供安全修复。发现问题后，请先在最新版上确认；旧版
可能要求先升级再修复。

| 版本 | 安全支持 |
| --- | --- |
| 1.13.x | 支持 |
| 1.12.x 及更早版本 | 仅在必要时评估 |

## 私下报告漏洞

请不要为以下问题创建公开 Issue：

- 未授权访问其他用户文件或个人 WebDAV；
- 分享令牌推导、绕过有效期或撤销失效；
- 任意文件读取、路径穿越、符号链接绕过；
- 认证绕过、权限提升或管理员边界突破；
- 数据库、JWT、远端存储凭据或备份密钥泄露；
- 可稳定复现的远程代码执行或拒绝服务。

优先使用 GitHub 仓库的 **Security → Report a vulnerability** 私下提交。请提供受影响
版本、部署方式、复现步骤、影响范围和你已采取的临时缓解措施。不要附带真实用户
文件、生产密钥、会话令牌或未脱敏数据库。

维护者会尽快确认报告、评估影响并协调修复和披露时间。在补丁可用前，请勿公开漏洞
细节或利用代码。

## 部署方责任

项目的默认防护不能替代正确运维。部署方应使用 HTTPS、强密码、独立密钥文件、
最小权限、防火墙、可靠的本地块存储和经过验证的离线备份，并及时更新宿主机、
Docker、反向代理和云粘贴。
